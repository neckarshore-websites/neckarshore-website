## P1 — High value

### P1-1 · `WebPage` entity is the homepage on all 18 non-homepage pages — `[CODE]`
**Tag:** VERIFIED-REAL (self-confirmed on omnopsis, phonesis, goldoni case study). **Source:** seo-schema.

On every non-homepage URL the `@graph` `WebPage` node is byte-identical:
```
@id = https://neckarshore.ai/#webpage  ·  url = https://neckarshore.ai/  ·  name = "neckarshore.ai — Software Development. Closer to Home."
```
Every product page tells Google's entity graph "I am the homepage." On detail pages there is then **no correct current-page entity at all** — precisely where the (correctly-placed) FAQPage citation value should anchor. Listing pages partially compensate via their `CollectionPage` block; detail + case-study pages do not.

- **Root cause:** the `@graph` block is generated once from a shared component, never parameterized per route.
- **Proposed change:** emit a page-scoped `WebPage` per route (`@id = .../products/<slug>#webpage`, correct `url`/`name`); keep `about`/`isPartOf`.
- **Failure check:** `curl -s …/products/omnopsis | …WebPage @id` → returns `…/products/omnopsis#webpage`, not `…/#webpage`.

### P1-2 · Product FAQ definitional answers too short & entity-unnamed for AI citation — `[CONTENT]`
**Tag:** VERIFIED-REAL (self-confirmed). **Source:** seo-geo P1 + seo-content P3 — **two independent agents converged.**

Primary "Was ist/macht X?" answers are 23-38 words vs the homepage's 120-122w, and several omit the subject entity:
- omnopsis "Was ist Omnopsis?" → 38w (names Omnopsis ✅)
- phonesis "Was ist die Phonesis Voicebank?" → **23w**, "Ein Archiv echter menschlicher Stimmen…" (does NOT name Phonesis)
- prod-or-pretend "Was macht Prod-or-Pretend?" → **26w**, "Es prüft…" (does NOT name the product)

Short, context-dependent answers lose the standalone-extractability that AI engines prefer.

- **Proposed change:** expand each product's primary definitional FAQ to a self-contained 100-140w answer that (a) opens `<ProductName> ist eine/ein <Kategorie>`, (b) states function, (c) names one concrete differentiator, (d) repeats the product name ≥2×. Leave correctly-short factual answers ("Konto nötig? → Nein. X ist anonym.") as-is. Content already exists in the page body — this is consolidation, not new writing.
- **Why P1:** this is the single highest-leverage lever for the Front-10 GEO/AI-citation goal.
- **Failure check:** re-parse FAQPage JSON-LD; primary answers ≥80w and start with the product name; readable standalone without the question.

## Rejected / not-defects (transparency — the verify-discipline payoff)

1. **❌ REJECTED — "AI-authorship disclosure missing on 5 pages" (seo-content P2-1).** FALSE-NEGATIVE. All 10 detail pages carry "Wie dieser Text entstand: …" in live SSR HTML. The skills pages render it inline with honest README-sourced wording ("KI-beschleunigt aus der README und Repo-Dokumentation … zusammengestellt, vom Gründer redigiert"); the agent grepped trafilatura-extracted text, which dropped the inline line as boilerplate. **4th false-positive of this exact class on the estate.**
2. **⚪ DOWNGRADED — "Case studies have no CTA" (seo-content P2-3) → P3-4.** A global header "Kennenlerntermin" button is present on every page; the real (minor) gap is only the *body* conversion-close.
3. **BY-DESIGN — "rauhut under-linked from /products hub" (seo-technical P3-2).** rauhut is the only Website without `featured: true` in `portfolio.ts`, so it is intentionally absent from the featured Top-3 teaser and lives on the sub-portal only. Not a bug — a design tradeoff (feature it, or accept the PageRank delta).
4. **ℹ️ NOT A DEFECT — FAQPage post-retirement (seo-technical P3-5).** Correct, valid, kept for AI-citation. Do not remove.
5. **Business decision, not a GEO gap:** CCBot/anthropic-ai/cohere-ai training-opt-out not declared in robots.txt.
